# 社科科研绘图提示词生成技能

## 概述

本技能用于生成高质量的社会科学学术绘图提示词，适用于理论框架图、概念模型图、服务机制图、社会生态系统结构图等。通过结构化的工作流程，将用户的论文内容转化为符合学术规范的文生图提示词。

## 工作流程

本技能采用五步工作流程，逐步引导用户生成专业的社科绘图提示词：

### 步骤1：收集内容素材
- **触发条件**：用户调用此技能时自动触发
- **用户交互**：要求用户提供论文内容素材
- **可接受格式**：论文摘要、理论框架描述、研究方法、核心概念、变量关系等
- **输出**：整理好的内容素材，准备用于下一步

### 步骤2：生成VISUAL SCHEMA
- **输入**：用户提供的内容素材
- **参考文件**：`references/prompt1.txt`（包含布局策略选择器和Schema生成规则）
- **用户交互**：
  1. 分析用户内容，基于prompt1.txt中的5种布局策略提供3种最适合的绘图类型选择
  2. 提供自定义输入框供用户选择其他类型
  3. 生成详细的VISUAL SCHEMA
- **输出**：结构化的VISUAL SCHEMA文档

### 步骤3：明确语言类型
- **输入**：VISUAL SCHEMA
- **参考文件**：`references/prompt2.txt`（包含艺术风格和执行指令）
- **用户交互**：
  1. 提供语言选择：中文（默认）、英文
  2. 提供自定义输入框供用户选择其他语言
  3. 确认最终语言类型
- **输出**：语言类型确认，准备生成最终提示词

### 步骤4：输出最终提示词
- **输入**：VISUAL SCHEMA + 语言类型
- **处理**：将VISUAL SCHEMA加载到prompt2.txt的`[[VISUAL SCHEMA]的全部内容]`部分
- **输出方式**：直接在对话框中以Markdown代码块格式输出完整提示词，用户可一键复制粘贴
- **注意**：不要创建单独的文件保存提示词，直接在对话中输出即可

### 步骤5：可选输出Mermaid/SVG代码
- **触发条件**：步骤4完成后自动触发
- **用户交互**：使用AskUserQuestion工具询问用户是否需要代码输出
- **选项**：
  1. 输出Mermaid代码
  2. 输出SVG代码
  3. 都输出
  4. 都不需要，结束任务
- **输出方式**：直接在对话框中以Markdown代码块格式输出，不要创建单独的文件
- **Mermaid**：基于VISUAL SCHEMA生成严格对照的Mermaid流程图代码
- **SVG**：基于VISUAL SCHEMA生成手绘风SVG矢量图代码

## 详细步骤说明

### 步骤1：收集内容素材

当用户调用此技能时，首先询问：

```
请提供您的论文内容素材，包括：
1. 研究主题和核心问题
2. 理论框架或概念模型
3. 主要变量或概念及其关系
4. 研究方法或分析框架
5. 任何其他相关的图表或结构描述

您可以提供文字描述、摘要或相关文件路径。
```

### 步骤2：生成VISUAL SCHEMA

1. **加载参考文件**：读取`references/prompt1.txt`了解布局策略和Schema生成规则
2. **分析用户内容**：基于5种布局策略分析用户内容：
   - Hierarchical Needs Pyramid（金字塔结构）
   - Central Governance Hub（中心治理枢纽）
   - Layered Service Ecology（分层服务生态）
   - Cyclic Social Interaction（循环社会互动）
   - Parallel Actor Network（平行主体网络）
3. **提供选择**：基于分析结果，提供3种最适合的绘图类型选择
4. **用户交互**：使用AskUserQuestion工具让用户选择
5. **生成Schema**：按照prompt1.txt的格式生成VISUAL SCHEMA

**选择示例**：
```
基于您的研究内容，我推荐以下3种绘图类型：

1. **中心治理枢纽结构** - 适合多主体协同、平台治理、智慧系统
2. **分层服务生态结构** - 适合公共服务体系、社会支持系统
3. **循环社会互动结构** - 适合行为反馈、参与式治理

请选择一种类型，或输入自定义类型：
```

### 步骤3：明确语言类型

1. **加载参考文件**：读取`references/prompt2.txt`了解艺术风格要求
2. **语言选择**：提供语言选项
3. **用户交互**：使用AskUserQuestion工具让用户选择语言
4. **确认**：记录最终语言类型

**选择示例**：
```
请选择文生图提示词的输出语言：

1. **中文**（默认）
2. **英文**

或输入自定义语言：
```

### 步骤4：输出最终提示词

1. **组合内容**：将VISUAL SCHEMA插入到prompt2.txt的`[[VISUAL SCHEMA]的全部内容]`位置
2. **语言处理**：根据选择的语言，确保所有文本标签使用正确语言
3. **格式化**：按照prompt2.txt的格式要求组织最终提示词
4. **输出方式**：直接在对话框中以Markdown代码块格式输出，格式如下：

````markdown
## 文生图提示词

将以下提示词复制粘贴到 Midjourney / DALL·E / Stable Diffusion 等模型：

```prompt
[完整提示词内容]
```
````

**重要**：不要创建单独的文件保存提示词。直接在对话框中输出Markdown格式，用户可一键复制。

### 步骤5：可选输出Mermaid/SVG代码

步骤4完成后，使用AskUserQuestion工具询问用户：

```json
{
  "question": "是否需要额外输出代码版本？",
  "header": "代码输出",
  "options": [
    {"label": "输出 Mermaid 代码", "description": "基于VISUAL SCHEMA生成Mermaid流程图代码，可粘贴到Mermaid Live Editor预览"},
    {"label": "输出 SVG 代码", "description": "基于VISUAL SCHEMA生成手绘风SVG矢量图代码，可直接在浏览器打开"},
    {"label": "都输出", "description": "同时输出Mermaid和SVG代码"},
    {"label": "都不需要", "description": "结束任务，只保留文生图提示词"}
  ],
  "multiSelect": false
}
```

根据用户选择，在对话框中以Markdown代码块格式输出对应代码：

**Mermaid输出格式**：
````markdown
## Mermaid 代码

将以下代码粘贴到 [Mermaid Live Editor](https://mermaid.live) 预览：

```mermaid
[Mermaid代码]
```
````

**SVG输出格式**：
````markdown
## SVG 代码

将以下代码保存为 `.svg` 文件，或直接在浏览器中打开预览：

```svg
[SVG代码]
```
````

**重要**：所有代码直接在对话框中输出，不要创建单独的文件。

## 注意事项

1. **内容质量**：用户提供的内容素材越详细，生成的提示词质量越高
2. **布局选择**：基于论文的实际逻辑选择最合适的布局策略
3. **语言一致性**：确保所有文本标签使用统一的语言
4. **学术规范**：生成的提示词必须符合社会科学学术绘图规范
5. **示例参考**：参考`assets/examples/`中的优质示例了解期望效果
6. **输出方式**：所有输出（提示词、Mermaid、SVG）直接在对话框中以Markdown格式输出，不要创建单独的文件
7. **Mermaid对照**：Mermaid代码必须严格对照VISUAL SCHEMA的Zone和Connection，可附加对照表
8. **SVG风格**：SVG代码应使用手绘风（rough.js），保持学术感但避免冰冷的工程感
